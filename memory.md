# 长期规则

## 1. 用户偏好

| 规则 | 说明 |
|------|------|
| 沟通语言 | 默认使用中文沟通 |
| 回答格式 | 先给结论，再给必要细节 |
| 输出要求 | 需要可执行结果，不要空泛描述 |
| 问题处理 | 优先直接修复，再解释原因 |

## 2. 命名规范

| 规则 | 说明 |
|------|------|
| Markdown标题 | 使用中文标题 `# 标题` |
| 变量与函数 | 优先语义，避免无意义缩写 |

## 3. 固定工作流

| 阶段 | 动作 |
|------|------|
| 改代码前 | 先检查现状（目录、关键文件、运行方式） |
| 改代码后 | 至少做一次基础校验（语法检查+本地运行验证） |
| 部署时 | 先Preview，再在用户明确要求后发Production |
| 发现错误时 | 先给可复现定位，再给最小修复，再回归验证 |

## 4. 亚马逊爬虫规则

| 规则 | 说明 |
|------|------|
| 默认站点 | amazon.it（用户指定意大利站） |
| 爬取数量 | 50个/批 |
| 相似度阈值 | 80%（可调） |
| 反爬处理 | 简单尝试，失败再处理代理/验证码 |
